Se realizo un CRUD (Crear, Leer, Actualizar, Eliminar) de suscripciones de usuarios a una tienda de música, no se uso ningún framework para realizarlo, pero si se utilizo el patrón MVC (Modelo-Vista-Controlador) que tiene las siguientes caracterisiticas:

Estructura basada en el patrón MVC:

Modelo: Maneja la lógica de datos, como la interacción con la base de datos para realizar operaciones CRUD (crear nuevas suscripciones, obtener información, actualizar datos y eliminar registros).

Vista: Se encarga de presentar la interfaz de usuario, mostrando las suscripciones y permitiendo interacciones como formularios para añadir o editar datos.

Controlador: Actúa como intermediario entre la Vista y el Modelo, gestionando las solicitudes del usuario y las respuestas apropiadas.


Interfaz de usuario dinámica y amigable apoyada con un framework de CSS Bootstrap el cual contiene lo siguiente:


Formularios para registrar y editar suscripciones de usuarios.

Listado de suscripciones actuales, organizado y fácil de navegar.

Mensajes de error según las acciones realizadas.

Gestión de suscripciones:

Creación: Permite que el usuario registre nuevas suscripciones con campos como nombre, correo electrónico, tipo de suscripción.

Lectura: Muestra una lista completa o detalles de cada suscripción individual.

Actualización: Modifica los detalles de una suscripción existente.

Eliminación: Borra registros específicos de suscripciones.

Interacción con base de datos:

Base de datos estructurada con la tabla de subscriptions el cual contiene los campos necesarios para registrar una nueva suscripción.

Validación y manejo de errores:

Validación en la creación de nuevas suscripciones, donde no se permite crear una suscripción si el campo email tiene un registro con el mismo dato suministrado (duplicidad).

Manejo de errores para mostrar mensajes claros al usuario en caso de problemas.

Modularidad y mantenibilidad:

Código organizado y modular, facilitando futuros cambios o ampliaciones.

Separación clara de responsabilidades entre el Modelo, la Vista y el Controlador.


Rutas:

La ruta principal es la siguiente /user/view/index.php, en esta ruta se listan las suscripciones actuales.

Ruta user/view/create.php, en esta ruta se crean nuevas suscripciones.

Ruta user/view/edit.php?id=1   En esta ruta dinamica, se modifican las suscripciones pasando por url el id de la suscripcion a modificar.
